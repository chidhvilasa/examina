# EXAMINA Technical Debt Register

| ID | Description | Reason Deferred | Priority | Risk | Target Release |
|----|-------------|-----------------|----------|------|----------------|
| TD-001 | RESOLVED — Branch protection status checks require CI to have run at least once | CI workflow now exists (`.github/workflows/ci.yml`) and reports on every PR; resolved in Phase 1. | LOW | LOW | v0.1.0 |
| TD-002 | With `strict` required status checks enabled, GitHub cannot evaluate a required context for a commit that has never been pushed — so direct pushes to `main` are permanently rejected, not just during bootstrap. All changes to `main`, forever, must land via a PR from a branch (CI runs on the `pull_request` event and reports against the PR's head SHA before merge). This is a standing operational characteristic of the Phase -1 branch protection configuration, not a one-time bootstrap hurdle that "resolves" after the first CI run. | Inherent to how GitHub evaluates required status checks against a specific commit SHA; not fixable without weakening branch protection (removing `strict` or the required contexts), which was explicitly requested in Phase -1. | LOW | LOW | N/A — accepted as the intended workflow |
| TD-003 | `LocalBridgeClient.analyze` and `RemoteBridgeClient.analyze` are Phase 1 stubs — the former returns fixed placeholder data, the latter always raises `BridgeError(code="BRIDGE_UNAVAILABLE")`. Neither invokes real PRISM or real HTTP. | Explicitly out of scope for Phase 1 per the bridge interface prompt; real implementations are blocked on Phase 2's report engine needing real PRISM output to test against. | LOW | LOW | v0.2.0 (Phase 2) |
| TD-004 | `BRIDGE_SPEC_v1.0.md`'s field-level type contract (`BridgeRequest.declared_mime_type`, `BridgeFact.fact_id`-less shape, `BridgeStatus` enum, etc.) was superseded by `BRIDGE_SPEC_v1.1.md` in Phase 1 without ever being implemented — `src/examina/bridge/types.py` was written directly against v1.1. See `docs/adr/ADR-0001-bridge-type-schema-amendment.md`. | The v1.0 field set, if implemented literally as originally frozen, would have shipped `BridgeFact`/`BridgeContradiction`/`BridgeHypothesis` types incompatible with the traceability chain in `EVIDENCE_TRACEABILITY_v1.0.md`; amended via ADR before any code was written against it. | LOW | LOW | N/A — documented, not pending work |
